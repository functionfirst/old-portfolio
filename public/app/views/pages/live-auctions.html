Real-time Auctions gulfplates.com

View demo system

The initial scope of the Gulf Plates project was to design and build a web application that could facilitate the sale of car number plates in real-time.

Features

  • Real-time Auctions (using Socket.io)
  • Auction management
  • User Registration & Management
  • Manual incremental bidding
  • Automated bidding

Summary

The final real-time auction system was based on the MEAN stack, using AngularJS for the front-end with a Node/ExpressJS back-end. The real-time nature of the app was facilitated by socket.io which allows for real-time bi-directional communication between web clients and the server.

Gulf Plates was a web application designed for the one off sale of live auction Live auction application that uses socket.io to instantly update connected client machines when someone submits a bid.

Auctions were set with start dates which counted down as the date/time got closer. Once the start time was reached, the NodeJS back-end would push an auction start request to each connected client allowing them to start bidding.

In cases where bidding was still going on the system allowed the administrator to set a time period which the auction was extended by each time a new offer was received. This left the auction system open ended but allowed each user time to make their bid while also preventing users 'sniping' a low bid in at the last second.

Bidding allowed the users to select their increment amount rather than a fixed bid amount. This allowed the user to quickly see the current bid and increment by the amount they selected.

Autobidding, for those users who had a clear idea of their maximum offer. Autobid allowed them to set a maximumum value for the auction. When other users bid, the autobid system would immediately counter the bid. In situations where there were multiple Autobids, placed the system would identify the highest Autobid and assign them as the current bid holder using the second highest Autobid as their offer.
